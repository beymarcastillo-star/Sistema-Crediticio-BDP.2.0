# 9. Estrategia de Implementación y Capacitación

## 9.1. Metodología de Implementación

El desarrollo del sistema se llevará a cabo mediante una metodología ágil, permitiendo una entrega progresiva de funcionalidades y una adaptación continua a los requerimientos del proyecto.

Se trabajará por iteraciones, donde cada fase incluirá análisis, desarrollo, pruebas y validación, garantizando que el sistema cumpla con los requerimientos funcionales y no funcionales establecidos.

Asimismo, se utilizarán entornos diferenciados para asegurar la calidad del software:

- **Entorno de Desarrollo:** Para la implementación y pruebas internas del sistema.
- **Entorno de Pruebas (Testing):** Para validar funcionalidades antes de su despliegue.
- **Entorno de Producción:** Para la operación final del sistema dentro del BDP S.A.M.


## 9.2. Equipo de Trabajo

El proyecto será desarrollado por un equipo técnico especializado en tecnologías web, conformado por:

- Desarrolladores Full-Stack encargados del desarrollo de la aplicación.
- Responsable del proyecto (Gestor de Pilas), encargado de la coordinación, seguimiento y control del avance.
- Usuarios clave (analistas de crédito), quienes participarán en la validación del sistema.

El equipo trabajará de forma colaborativa, asegurando la correcta interpretación de los requerimientos y el cumplimiento de los objetivos del proyecto.


## 9.3. Estrategia de Capacitación

Se implementará una estrategia de capacitación basada en el enfoque "aprender haciendo", orientada a los usuarios finales del sistema.

Para ello se contemplan las siguientes acciones:

- Uso de un entorno **SandBox (ambiente de pruebas)** donde los usuarios podrán interactuar con el sistema sin afectar datos reales.
- Capacitación progresiva por módulos del sistema (registro, evaluación, reportes, etc.).
- Elaboración de documentación tipo guía o manual de usuario.
- Sesiones prácticas para reforzar el uso del sistema en escenarios reales.

Esta estrategia permitirá que los usuarios adquieran experiencia directa en el uso de la plataforma, facilitando su adopción.


## 9.4. Control de Cambios

Se establecerá un control de cambios riguroso para garantizar la estabilidad del sistema y la correcta gestión de modificaciones.

Para ello se considerará:

- Separación de ambientes (Desarrollo, Pruebas y Producción).
- Validación previa de cambios antes de su despliegue en producción.
- Registro y seguimiento de cambios realizados en el sistema.
- Gestión de versiones mediante repositorios de control de código.

Este enfoque permitirá mantener la integridad del sistema, minimizar riesgos y asegurar una evolución controlada de la plataforma.
